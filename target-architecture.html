Target Architecture Layers

Network Foundation

VPC, public/private subnets, routing, security groups, load balancing and network segmentation.

Application Platform

ECS, EC2 and Auto Scaling support phased migration, containerised services and scalable application hosting.

Data Platform

RDS SQL Server, S3 and AWS Backup provide managed data hosting, durable storage and recovery governance.

Security & Access

IAM, Secrets Manager, WAF, CloudTrail and least-privilege controls improve access governance and auditability.

Observability Layer

CloudWatch metrics, logs, alarms and dashboards provide proactive monitoring and faster root cause analysis.

AI Operations Intelligence

AI-assisted troubleshooting uses operational data, historic tickets, logs and runbooks to guide investigations.

Key AWS Service Decisions

Each AWS service is selected to address a specific operational or architectural concern from the current environment.

Amazon VPC

Provides network isolation, subnet design, segmentation and controlled communication between application and data layers.

Application Load Balancer

Improves availability by distributing traffic across application workloads and supporting multi-AZ resilience patterns.

Amazon ECS

Supports modernised container workloads and allows services to scale more flexibly as the platform evolves.

Amazon EC2

Supports transitional workloads that may need virtual machine compatibility during migration from on-premises systems.

Amazon RDS SQL Server

Reduces database operational overhead while supporting managed backups, patching options and Multi-AZ availability.

SQS & SNS

Decouple operational workflows, buffer work between services and support event-driven alerting and integration patterns.

AWS Secrets Manager

Reduces shared credential risk by centralising secrets, supporting rotation and improving operational access governance.

AWS CloudTrail

Provides audit visibility into API activity and supports investigation, compliance and governance requirements.

Amazon CloudWatch

Enables metrics, logs, alarms and dashboards to support proactive monitoring and operational intelligence.

How the Target State Addresses Current Risks

Recurring Outages

Multi-AZ design, load balancing, auto scaling and improved observability reduce service impact and improve availability.

Manual Failover

Managed services, documented runbooks, backup policy and recovery design reduce reliance on manual intervention.

Monitoring Gaps

CloudWatch dashboards, alarms, logs and operational alerts improve visibility and support earlier detection.

Credential Risk

Secrets Manager, IAM roles and least privilege reduce shared password risk and improve auditability.

Messaging Reliability

SQS and SNS provide decoupled messaging patterns that improve resilience and reduce workflow disruption.

DR Weakness

AWS Backup, Multi-AZ services, tested runbooks and defined recovery objectives strengthen business continuity.

Architecture Trade-Offs

ECS vs EKS

ECS is selected initially to reduce operational complexity while still supporting containerised workloads. EKS may be considered later if Kubernetes portability, multi-cloud patterns or advanced orchestration become stronger requirements.

EC2 Transitional Workloads

EC2 remains part of the target architecture to support workloads that cannot immediately be containerised. This allows phased migration without forcing unnecessary refactoring too early.

Managed Database Approach

RDS SQL Server reduces operational overhead but requires careful licensing, performance sizing, backup policy and migration planning.

AWS-First, Not AWS-Only

The target is AWS-first for portfolio focus and implementation clarity, while still considering future hybrid, multi-cloud resilience, portability and backup strategies where appropriate.

Future AI Operations Layer

The AI operations layer is designed to extend observability and support workflows by helping engineers investigate issues faster, correlate incidents, retrieve documentation and follow structured runbooks.

Knowledge Retrieval

Search operational documentation, known issues, past incidents, architecture notes and support runbooks.

Log & Incident Analysis

Use AI to summarise logs, correlate alerts and identify likely root causes or affected services.

Guided Troubleshooting

Suggest investigation steps, escalation paths, rollback options and validation checks based on operational context.

Auto-Ticketing

Convert alerts or detected issues into structured tickets with impact summary, affected systems and recommended actions.

Architecture Q&A

Allow users to ask why services were chosen, how risks are mitigated and what trade-offs were considered.

Continuous Improvement

Use recurring incidents and operational trends to improve architecture, runbooks and platform reliability.
