Phase 3: Modernise

Application & Operations Modernisation

Modernising the fulfilment platform beyond migration by improving application architecture, observability, release capability, resilience, security and operational support maturity.

Modernisation Objective

Phase 3 focuses on improving how the platform is built, deployed, monitored, recovered and operated. The goal is not only to host workloads on AWS, but to reduce operational fragility and create a more scalable, resilient and supportable architecture.

Application Modernisation

Move from legacy application servers to containerised services running on ECS Fargate across multiple Availability Zones.

Operational Modernisation

Introduce centralised logging, dashboards, alarms, runbooks and automated operational workflows.

Delivery Modernisation

Prepare the platform for CI/CD, controlled releases, automated validation and safer rollback patterns.

Current to Modernised State

The modernisation phase maps legacy platform components to managed AWS services and improved operational patterns.

Current State Modernised Target Why It Matters
APP-1 / APP-2 application server pair ECS Fargate application services across AZ A and AZ B Reduces server management overhead and improves scaling, availability and deployment flexibility.
SQL Server cluster supporting operational and reporting workloads Amazon RDS SQL Server Multi-AZ with reporting replica Improves availability and separates reporting load from core operational transactions.
RabbitMQ messaging broker Amazon MQ with Dead Letter Queue and EventBridge integration Improves messaging resilience, failure handling and operational event routing.
Fragmented log search and manual investigations CloudWatch Logs, OpenSearch Service and AI Operations Assistant Improves incident investigation, knowledge retrieval and support efficiency.
Manual patching and recovery Systems Manager Runbooks and Patch Manager Reduces manual intervention and improves repeatability of operational processes.
Shared passwords and inconsistent secrets handling Secrets Manager, KMS and IAM least privilege Improves access governance, auditability and security posture.

Application Platform Modernisation

The application platform is decomposed into containerised services aligned to fulfilment responsibilities.

Host Message Service

Handles external fulfilment messages and integration requests from upstream business systems.

Order Fulfilment Service

Coordinates fulfilment workflow execution, order state and operational task progression.

Stock Allocation Service

Manages stock allocation logic while reducing coupling with wider fulfilment workflows.

Task Management Service

Controls operational task creation, assignment and progression through fulfilment processes.

Workflow Service

Manages orchestration logic, workflow progression and fulfilment process coordination.

Robot Integration Service

Provides a controlled integration boundary between fulfilment services and the robot control system.

ECS Fargate was selected because it supports containerisation without requiring the team to manage EC2 cluster capacity. This reduces operational overhead while still providing resilience, scaling and a clear path toward CI/CD-enabled deployments.

Observability Modernisation

Observability is introduced as a core architecture capability rather than an afterthought. This directly addresses the current state problem of reactive support and limited proactive monitoring.

CloudWatch Logs

Centralises application, platform and operational logs from ECS services, RDS, messaging and support workflows.

CloudWatch Alarms

Triggers alerts for queue depth, service health, database performance, error rates and availability signals.

CloudWatch Dashboards

Provides operational teams with a single view of service health, incident indicators and platform performance.

OpenSearch Service

Enables searchable operational logs and investigation across application events, errors and incident patterns.

SNS Notifications

Routes alerts to support teams so incidents are detected earlier and response is more structured.

Systems Manager Runbooks

Provides repeatable operational actions for investigation, recovery, patching and support procedures.

Resilience & Recovery Modernisation

Multi-AZ Application Services

ECS services run across private application subnets in multiple Availability Zones to reduce single-AZ failure risk.

Managed Database Failover

RDS SQL Server Multi-AZ reduces reliance on manual SQL failover and improves database availability.

Reporting Separation

Reporting replica separates read-heavy reporting workloads from operational transaction processing.

Managed Messaging Resilience

Amazon MQ, DLQ handling and operational alerts reduce the impact of queue backlogs and failed messages.

Backup & Restore Testing

AWS Backup creates a governed backup approach with restore testing built into operational readiness.

Operational Runbooks

Repeatable recovery procedures reduce dependency on individual knowledge and manual intervention.

Release & DevOps Modernisation

The modernised platform creates the foundation for controlled CI/CD, safer deployments and standardised release governance.

Container Image Build

Application services can be packaged as container images and stored in a container registry such as Amazon ECR.

Automated Deployment

GitHub Actions or AWS CodePipeline can deploy validated container versions to ECS Fargate services.

Controlled Rollback

Release strategy can support rollback to previous task definitions if validation or production checks fail.

Environment Promotion

Changes can move through development, test, staging and production with consistent checks and approvals.

The current state has manual hotfix and patching risk. The modernised target introduces a clearer path toward automated release pipelines, validation gates and safer rollback while keeping the initial design realistic for a phased transformation.

Security Modernisation

IAM Least Privilege

Access is controlled using roles, scoped permissions and separation of operational responsibilities.

Secrets Manager

Application and database credentials are stored securely instead of being shared or manually managed.

KMS Encryption

Encryption keys protect sensitive data, backups, secrets and operational artefacts.

CloudTrail

API activity is captured for auditing, investigation and accountability.

GuardDuty & Security Hub

Threat detection and security posture findings are centralised for operational review.

AWS Config

Configuration changes and compliance posture are tracked across the AWS environment.

Modernisation Outcomes

Reduced Server Dependency

Legacy application servers are replaced with managed container execution through ECS Fargate.

Improved Operational Visibility

Support teams gain logs, alarms, dashboards, search and runbooks for structured investigation.

Improved Resilience

Multi-AZ deployment, managed database failover, backup governance and messaging resilience reduce outage impact.

Safer Change Delivery

Containerised services create a foundation for CI/CD, validation and controlled rollback.

Stronger Security Posture

Secrets, access, encryption, audit logging and compliance controls are built into the platform design.

Prepared for AI Operations

Logs, runbooks, known issues and support knowledge are structured for the AI Operations layer.
